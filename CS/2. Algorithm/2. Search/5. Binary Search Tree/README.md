# 이진 탐색 트리 (Binary Search Tree, BST)

이진 탐색 트리 (Binary Search Tree)는 이진 트리(Binary Tree)의 한 종류로, 이진 탐색(Binary Search)을 빠르게 수행하기 위해 만들어진 자료구조이다.

BST는 다음과 같은 특징을 갖는다.

- 모든 노드는 최대 두 개의 자식 노드를 가질 수 있다.

- 왼쪽 자식 노드는 현재 노드보다 작은 값을 가지며, 오른쪽 자식 노드는 현재 노드보다 큰 값을 가진다.

- 중복된 값을 가지는 노드는 존재하지 않는다.

BST에서 이진 탐색을 수행할 때, 루트 노드에서부터 시작해서 탐색하고자 하는 값을 찾을때까지 왼쪽 또는 오른쪽 자식 노드로 이동한다. 따라서 BST에서 탐색 시간의 평균적인 시간복잡도는 O(log n)이 된다.

BST는 삽입, 삭제, 탐색 연산에 대해서도 평균적인 시간복잡도가 O(log n)이다. 하지만 최악의 경우, 모든 노드가 한 쪽으로 치우쳐 있는 편향 이진 트리(skewed binary tree)의 경우, BST의 성능은 O(n)이 될 수 있다. 따라서 이를 방지하기 위해 AVL 트리나 레드-블랙 트리 같은 균형 잡힌 이진 탐색 트리를 사용하기도 한다.

## 클래스형 코드와 함수형 코드의 차이

이진 탐색 트리에서는 클래스형 코드와 함수형 코드 중 성능 면에서 큰 차이가 없다. 따라서 코드의 성능보다는 가독성과 유지보수성이 중요한데, 함수형 프로그래밍은 사이드 이펙트가 없는 순수 함수를 작성하는 것을 중요시하기 때문에 이 측면에서는 함수형 코드가 더 좋은 결과를 보이기도 한다.

또한 함수형 코드는 재귀적인 구조를 띄기 때문에, 이진 탐색과 트리와 같은 자료 구조와 자연스럽게 어울리기도 한다.
